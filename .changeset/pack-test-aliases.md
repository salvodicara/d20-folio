Make the content-pack test suites location-independent under the split-repo composition: the pack now composes in as a gitignored symlink to the sibling private-repo checkout, so pack tests can no longer reach public-root helpers via physical `../../../` escapes (vitest realpaths the files into the content repo and every escape resolves against the wrong root). Root-anchored `@tests/*` / `@scripts/*` aliases are wired wherever `@pack` is (vitest + vite alias maps, tsconfig paths — always valid, they point at the public root in both build modes), the vitest lanes resolve with `preserveSymlinks: true` so pack modules keep their in-root path (bare imports anchor at this repo's node_modules — test-lane only, the production build keeps realpath resolution), the dev server allows the pack's real directory via `server.fs.allow` (`fsAllowRoots()`, the one content-pack-mode seam), and `pnpm lint` names the pack glob explicitly (eslint's `.` traversal does not follow symlinks — the pack suites had silently dropped out of the lint gate). Ride-alongs: the composition symlink is gitignored, the spent public-snapshot-builder changeset is dropped, the open-sourcing epic is marked shipped in PROGRESS.md, and the local-composition recipe is documented (CONTRIBUTING + CLAUDE.md + ARCHITECTURE).
